## 4. 유량제어 (Rate Limiting)

아래는 **DB증권 서버가 적용하는 제한**이며, 초과 시 `IGW00201`(호출 거래건수 초과)이 반환됩니다.

**`dbsec_sdk` 는 이 제한을 클라이언트에서 자동으로 지킵니다.** REST 호출 직전에 **2-tier 유량제어**
(앱 20TPS + 엔드포인트별 TPS)를 적용해, 동시/대량 호출에서도 `IGW00201` 을 사전 차단합니다.
엔드포인트별 TPS 의 단일 소스는 [`api_support_matrix.md`](api_support_matrix.md) 의 `TPS` 컬럼이며,
`dbsec_sdk/rate_limiter.py` 가 이를 읽어 적용합니다. 비활성화는 `DBSecClient(..., rate_limit=False)`.

### REST API

| 구분 | 제한 | 설명 |
|------|------|------|
| **앱(계좌)별** | 20 TPS | 초당 20건 — SDK 글로벌 리미터가 자동 적용 |
| **토큰 발급/폐기** | 1 TPM | 1분에 1건 — 파일 캐시(24h 토큰 재사용)로 사실상 회피 |
| **API별** | 개별 | 주문 10TPS, 잔고조회 2TPS 등 — SDK 가 엔드포인트별 리미터로 자동 적용 |

### WebSocket

| 구분 | 제한 | 설명 |
|------|------|------|
| **세션 수** | 2세션 | 앱(계좌)당 최대 2개 |
| **종목 수** | 50종목/세션 | 세션당 50종목 — 초과 시 SDK 가 `RateLimitError` 발생 |
| **연결 빈도** | 6 TPM | 1분당 6건 (서버 정책) |
| **연결 후 Send** | 10초 이내 | 미전송 시 연결 해제 |
| **송수신 딜레이** | 100ms | `DBSecWebSocket` 이 안정화 딜레이 자동 적용 |

### 대량/동시 호출 (SDK 자동 페이싱)

SDK 가 유량제어를 자동 적용하므로, 직접 `sleep` 을 넣을 필요 없이 그대로 호출하면 됩니다.
동시 호출은 `asyncio.gather` 로 묶으면 유량제어를 지키면서 가장 빠르게 끝납니다.

```python
import asyncio

# 여러 종목을 동시에 — 같은 엔드포인트는 그 API 의 TPS 만큼 자동 페이싱
resps = await asyncio.gather(*[
    client.apis.kr_stock_quote.kr_stock_inquire_price(
        InputCondMrktDivCode="J", InputIscd1=code)
    for code in codes
])
```
